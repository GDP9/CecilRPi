Cecil

CECIL assembly language grammar definition This grammar constitutes all the lexer rules employed by CECIL and corresponding parsing rules. It is a version of the CECIL Language originally developed by David Argles. it contains 39 simple instructions and user input datafields and labelfields. It catches the relevant errors thrown by the lexer and parser. The language used is Java.   • program Top

    Description:

    CECIL sim40 LL(k) grammar

    Lexer : Tokens (Reserved Keywords) => all-instructions

    A valid program

    • may begin with an optional '.start' label
    • is constituted by multiple statements followed by optional assigments

    EBNF notation:

    program ::= '.start'? mnemonicdata statement* assignment*

    Description:

    A valid statement

    • may begin with an optional label
    • is constituted by multiple mnemonicdata

    EBNF notation:

    statement ::= ('.' labelfield)? mnemonicdata

  • assignment Top

    Description:

    A valid assignment

    • may begin with an optional label
    • is constituted by multiple 'insert' instructions followed by integer data values

    EBNF notation:

    assignment ::= ('.' labelfield)? 'insert' value

  • mnemonicdata Top

    Description:

    A valid mnemonicdata is

    • a binary instruction followed by a datafield
    • or a unary instruction

    EBNF notation:

    mnemonicdata ::= (binaryinstruction datafield | unaryinstruction)
